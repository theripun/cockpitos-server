To set up Cloudflare R2 for your gallery app, you need to configure CORS (Cross-Origin Resource Sharing) on your R2 bucket.

This is required because your 
gallery.tsx
 component performs direct file uploads (via PUT requests) and downloads (via GET request signed URLs) from the browser to R2.

1. Configure CORS in Cloudflare R2
Go to your Cloudflare Dashboard -> R2 -> Select your bucket (cockpit-preview) -> Settings -> CORS Policy.

Add the following policy to allow your local development and production domains to access the bucket directly:

json

[
  {
    "AllowedOrigins": [
      "http://localhost:3000",
      "https://cockpit.ripun.site",
      "https://cognode.a2.cockpit.run"
    ],
    "AllowedMethods": [
      "PUT",
      "GET",
      "HEAD",
      "DELETE"
    ],
    "AllowedHeaders": [
      "Content-Type"
    ],
    "ExposeHeaders": [
      "ETag"
    ],
    "MaxAgeSeconds": 3000
  }
]


2. Verify Server Environment Variables
I checked your 
server/.env
 file, and your R2 credentials appear to be already set up:

R2_ACCOUNT_ID: Set
R2_ACCESS_KEY_ID: Set
R2_SECRET_ACCESS_KEY: Set
R2_BUCKET: Set to cockpit-preview
3. Summary of Flow
Upload: The frontend requests a "signed URL" from your server.
Direct Transfer: The browser uses this URL to PUT the file directly to R2 (this is why CORS is needed).
Download: The frontend requests a signed URL to GET the image for previewing.
Note: You do not need to enable "Public Access" for the bucket since you are using signed URLs, which securely grant temporary access to private objects.
